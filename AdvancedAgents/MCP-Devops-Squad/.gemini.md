# MCP-DevOps-Squad

Production-ready Multi-Agent SRE System using Model Context Protocol (MCP) and Google Agent-to-Agent (A2A) patterns.

## Project Standards

- **Architecture:** Modular Python-based structure.
- **Communication:** Pydantic models for all inter-agent A2A hand-offs to ensure type safety.
- **MCP Integration:** All hardware/service access (Docker, GitHub, Filesystem) must go through MCP servers.
- **Security:**
    - No hardcoded credentials; use environment variables.
    - Janitor-Agent requires a "Human-in-the-loop" flag or "Validator" sign-off for destructive commands (`rm`, `kill`, etc.).
- **Logging:** Structured JSON logging in `/logs/` for audit trails.
- **Testing:** Every agent must have corresponding unit tests in `/tests/`.

## Directory Structure

- `src/agents/`: Individual class definitions for Monitor, Debugger, and Janitor agents.
- `src/mcp/`: Configuration and connection logic for MCP servers.
- `src/orchestrator/`: Lead-SRE logic handling A2A delegation loops.
- `config/`: YAML files for agent personas and threshold triggers.
- `logs/`: Structured JSON audit logs.
- `tests/`: Project test suite.
